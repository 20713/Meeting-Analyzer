会议时间：2023年10月25日 14:00-16:00
会议地点：301会议室
参会人员：张三（产品经理）、李四（开发主管）、王五（测试主管）、赵六（UI设计师）
主持人：张三

会议主题：Meeting Analyzer 产品需求评审

会议内容：
1. 需求背景介绍
张三介绍了 Meeting Analyzer 的背景，旨在通过 AI 技术帮助团队快速分析会议纪要，提取待办事项和决策点。目前痛点是人工整理会议纪要效率低，容易遗漏重要信息。

2. 核心功能讨论
- 自动生成会议摘要：李四建议使用 LLM 进行摘要生成，支持 Markdown 格式。
- 待办事项提取：王五提出需要明确责任人和截止时间，否则难以追踪。大家一致同意。
- 风险识别：赵六补充，系统应能识别会议中提到的潜在风险，并高亮显示。

3. 技术选型
- 前端：使用 Gradio 快速构建原型。
- 后端：Python + OpenAI SDK。
- 模型：选用 Qwen-Plus 模型，性价比高。

4. 决策与分工
- 确定采用 Gradio + Qwen-Plus 的技术栈。 (决策)
- UI设计由赵六负责，本周五前出初稿。
- 后端接口由李四负责，下周三前完成。
- 测试用例由王五负责，配合开发进度。

5. 风险
- API 调用成本可能超出预算，需监控 Token 使用量。
- 数据隐私问题，需确保用户上传的文件不被用于模型训练（使用企业级API或本地部署模型）。

6. 下一步计划
- 下周一进行 UI 评审。
- 下周五进行 Alpha 版本内部测试。
